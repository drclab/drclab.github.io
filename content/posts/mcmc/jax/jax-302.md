Welcome to JAX 302. This post is based on Section 2 (Overview) of *Probabilistic Programming with Vectorized Programmable Inference*, and turns that overview into a practical playbook for JAX users.

Section 2 uses one running example—polynomial regression with outliers—to show four escalating ideas:

1. Vectorize the generative model.
2. Vectorize programmable inference interfaces.
3. Improve model robustness with stochastic branching.
4. Improve inference quality with custom, composable algorithms.

## 1. Running example: polynomial regression as a generative program

The overview starts with a quadratic curve model where parameters $(a, b, c)$ are random, then each observation is generated as noisy output of

$$
y = a + bx + cx^2 + \epsilon.
$$

The key design choice is to represent model execution as a **trace** of named random choices. That trace can later be scored, constrained, and edited by inference algorithms.

## 2. First vectorization: model-level `vmap`

The first `vmap` win is straightforward: conditionally independent datapoints can be generated in parallel. Instead of looping in Python, map the point generator over an input vector of $x$ values.

```python
import jax
import jax.numpy as jnp

def point(x, a, b, c, sigma=0.2):
    mean = a + b * x + c * x**2
    return mean + sigma * jax.random.normal(jax.random.PRNGKey(0), shape=())

def curve(xs, a, b, c):
    return jax.vmap(lambda x: point(x, a, b, c))(xs)
```

Conceptually, this keeps the model the same but changes execution from scalar to batched array kernels.

## 3. Second vectorization: inference-level `vmap`

Section 2 then highlights a deeper point: in programmable inference, we repeatedly call interface methods like:

- `simulate`: draw a trace from a model/proposal.
- `assess`: score that trace under a target density.

If a single-particle importance sampler is written in terms of these interfaces, `vmap` can lift it into many-particle importance sampling by changing only batch dimensions.

That gives a practical scaling law: increase particle count $N$ until device memory saturates; runtime often stays close to flat over a useful range while approximation quality improves.

## 4. Vectorized traces: struct-of-arrays mindset

Section 2 emphasizes that vectorized execution should preserve trace structure while changing value shapes.

- Single-run trace fields: scalar or low-rank arrays.
- Vectorized trace fields: leading batch dimension(s).

This is effectively a struct-of-arrays representation, which fits JAX transformations better than object-heavy array-of-struct layouts.

## 5. Robustness via stochastic branching

The overview extends the observation model to handle outliers with a latent binary flag per datapoint:

- If outlier: sample from a broad distribution.
- Else: sample from the polynomial noise model.

This is stochastic branching inside the generative process. The model becomes more realistic, but posterior geometry gets harder and naïve inference can underfit.

## 6. Better inference via programmable composition

Section 2’s key engineering lesson: when model complexity rises, change the inference program, not just particle count.

The paper’s example combines:

- Gibbs updates for discrete outlier indicators.
- HMC updates for continuous curve parameters.

The Gibbs step itself is vectorized across datapoints when conditional independencies allow it. This creates a custom hybrid MCMC routine that is both expressive and accelerator-friendly.

## 7. Interface-driven design pattern

A useful takeaway for your JAX projects is to write inference around a small interface surface:

- Sample unconstrained traces.
- Score traces.
- Generate constrained traces.
- Update existing traces.

Once these are explicit, vectorization and algorithm composition become mechanical rather than ad hoc.

## 8. What to carry into your own code

From Section 2, the practical strategy is:

1. Start with a clear generative model and named latent structure.
2. Batch conditionally independent model pieces with `vmap`.
3. Batch inference particles/chains with the same transform.
4. Introduce branching for realism when needed.
5. Upgrade inference algorithms (e.g., hybrid kernels) when accuracy stalls.

In short: **scale by vectorizing both modeling and inference, then adapt inference design to model complexity.**
