Welcome to Rust 108-3! Hash maps are Rust's way of storing key-value pairs, allowing efficient lookups by key rather than index. The type `HashMap<K, V>` uses a hashing function to place data in memory. In this post, we'll cover creating hash maps, accessing values, handling ownership, updating entries, and understanding hashing functions.

## Creating a New Hash Map

To create a hash map, import it from the standard library and use `new()`:

```rust
use std::collections::HashMap;

let mut scores = HashMap::new();

scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Yellow"), 50);
```

Hash maps require all keys to be the same type and all values to be the same type. Unlike vectors, there's no built-in macro for construction.

## Accessing Values in a Hash Map

Use the `get` method to retrieve values:

```rust
let team_name = String::from("Blue");
let score = scores.get(&team_name).copied().unwrap_or(0);
```

`get` returns `Option<&V>`, so we handle it with `copied()` to get `Option<i32>` and `unwrap_or(0)` for a default.

Iterate over key-value pairs:

```rust
for (key, value) in &scores {
    println!("{key}: {value}");
}
```

Iteration order is arbitrary.

## Hash Maps and Ownership

For `Copy` types like `i32`, values are copied. For owned types like `String`, values are moved:

```rust
let field_name = String::from("Favorite color");
let field_value = String::from("Blue");

let mut map = HashMap::new();
map.insert(field_name, field_value);
// field_name and field_value are now invalid
```

You can insert references, but they must remain valid as long as the map exists.

## Updating a Hash Map

### Overwriting a Value

Inserting the same key replaces the value:

```rust
let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Blue"), 25);
println!("{scores:?}");  // {"Blue": 25}
```

### Adding a Key and Value Only If a Key Isn't Present

Use `entry` and `or_insert`:

```rust
let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);

scores.entry(String::from("Yellow")).or_insert(50);
scores.entry(String::from("Blue")).or_insert(50);

println!("{scores:?}");  // {"Yellow": 50, "Blue": 10}
```

### Updating a Value Based on the Old Value

Combine `entry` with dereferencing for updates:

```rust
let text = "hello world wonderful world";

let mut map = HashMap::new();

for word in text.split_whitespace() {
    let count = map.entry(word).or_insert(0);
    *count += 1;
}

println!("{map:?}");  // {"world": 2, "hello": 1, "wonderful": 1}
```

## Hashing Functions

By default, `HashMap` uses SipHash for security against DoS attacks. For performance-critical code, you can specify a different hasher implementing `BuildHasher`. Check crates.io for alternatives.
